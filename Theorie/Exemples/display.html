inline, block et inline-block

Je vais vous montrer dans les exemples suivants les différences entre inline, block et inline-block.

  • inline
  • block
  • inline-block

1. Inline

Dans un display de type inline, l'element ne pousse pas les éléments à la ligne, la hauteur et la largeur ne sont pas variables.

Dans l'exemple plus bas, nous utiliserons du css et du html.

CSS

.element {
    background: dodgerblue;
    color: snow;
    width: 100px;
    height: 100px;
    padding: 5px;
    border: 1px solid gainsboro;
}

HTML

    <div>
    Je suis un stagiaire <span class="element">très studieux.</span>. J'aime beaucoup <div class="element">relire mon cours</div> le jour avant pour mieux le comprendre.
    </div>
Résultat
Je suis un stagiaire très studieux.. J'aime beaucoup
relire mon cours
le jour avant pour mieux le comprendre.

Modifions la classe element et ajoutons: display: inline.

.element {
    display: inline;
    background: dodgerblue;
    color: snow;
    width: 100px;
    height: 100px;
    padding: 5px;
    border: 1px solid gainsboro;
}
Résultat
Je suis un stagiaire très studieux.. J'aime beaucoup
relire mon cours
le jour avant pour mieux le comprendre.

On constate que notre div ne retourne plus à la ligne grâce à inline et sa taille et sa largeur ne sont pas variables.

2. block

On va modifier notre classe .element et mettre à la place display: block

.element {
    display: block;
    background: dodgerblue;
    color: snow;
    width: 100px;
    height: 100px;
    padding: 5px;
    border: 1px solid gainsboro;
}
Résultat
Je suis un stagiaire très studieux.. J'aime beaucoup
relire mon cours
le jour avant pour mieux le comprendre.

3. inline-block

On va modifier notre classe .element et mettre à la place display: inline-block.

.element {
    display: inline-block;
    background: dodgerblue;
    color: snow;
    width: 100px;
    height: 100px;
    padding: 5px;
    border: 1px solid gainsboro;
}
Résultat
Je suis un stagiaire très studieux.. J'aime beaucoup
relire mon cours
le jour avant pour mieux le comprendre.

On voit donc qu'inline-block va disposer sur une même ligne nos éléments appelant notre classe. On aura des blocks de hauteur/largeur variables sur une seule ligne sans qu'ils poussent les autres éléments à la ligne suivante.
